If you’ve checked user profiles on GitHub, you may have noticed some are personalized with images, trophies, skill badges, and activity trackers. This feature was introduced in July 2025, allowing users to showcase skills and badges on their profile.

Today, I want to share how I integrated my **Wakatime dashboard stats** into my GitHub profile. Wakatime is an IDE plugin that tracks your coding activity, including time spent, projects worked on, and languages used. It’s a great tool for developers who want insights into their productivity.

## 1: Create a Wakatime Account

- Sign up on [Wakatime](https://wakatime.com/signup).
- Install and integrate the plugin with your IDE by following the [instructions here](https://wakatime.com/plugins).
- Wakatime supports most popular IDEs. If yours isn’t listed, you can reach out to their team.

## 2: Access and Share Your Dashboard

1. Go to your [Wakatime dashboard](https://wakatime.com/dashboard).
2. Click on the **Share** option.
3. Select **Embeddable charts** from the available options.

## 3: Customize Your Chart

- Adjust the chart style, timeframe, and customization options.
- Once satisfied, click **Embed code**.
- Copy the generated code.

## 4: Add to Your GitHub Profile

- Open your **GitHub profile README.md** file.
- Paste the embed code where you want your Wakatime stats to appear.
- Save and commit the changes.

You can check out a working example on [my GitHub profile](https://github.com/Osalumense).
